# Guide 10 — Generating reports

Navigate to **Reports** in the sidebar, or click **Report** on any tenant dashboard.

TrustM365 generates **tenant-level reports** covering a single tenant over a defined date range. Reports are always rendered in light mode — clean, professional, and suitable for printing, PDF, and formal client delivery.

---

## The Generate Report flow

### Step 1 — Configure

In the report modal, set:

- **Tenant** — which tenant to report on
- **Date range** — the period to cover. Quick presets: 7d, 30d, 90d

### Step 2 — Review the preview

TrustM365 immediately fetches a live preview of what the report will contain. The modal is split into three zones:

**Left — section tabs**

| Section | Preview shows |
|---|---|
| Executive Summary | Health score, drift count, remediation count, outstanding items |
| Drift History | Actual property changes with before/after values |
| Remediation Log | Auto vs manual restores, success/failure breakdown |
| Baseline Coverage | Per-area monitoring status |
| Current Configuration State | Users, groups, apps, device counts |

**Centre — live preview**

Shows the real data from the database for the selected tenant and date range. What you see is exactly what will appear in the generated report.

**Right — MSSP commentary**

A text area for each section. Commentary is optional — leave it blank to omit, or add context to appear as a highlighted callout in that section. When a company name is set in MSSP Settings, the callout label reads **"[Your Company] Commentary"** instead of "MSSP Commentary".

### Step 3 — Generate

Click **Generate & View Report**. The report opens immediately in the full-screen viewer.

---

## Report sections

### Executive Summary

- Company logo / name (from MSSP Settings, or "TrustM365" if not configured)
- Tenant name and reporting period
- Stat summary: Drift Events · Remediated · Outstanding · Areas Monitored
- Plain-English summary sentence
- MSSP commentary callout (if added)

### Drift History

- Table: Area / Resource / Property / Baseline value / Detected value / Detected at
- Drift events are deduplicated per area — each area counts as one event regardless of how many syncs confirmed it

### Remediation Log

- Stats: Total / Succeeded / Failed / Auto-restored / Manual
- Per-action table: Area / Resource / Property / Trigger / Result / Timestamp

### Baseline Coverage

Grouped by product category:
- **Microsoft Entra ID**
- **Microsoft Intune — Policy Management**
- **Microsoft Intune — Endpoint Security**

Summary strip: Total Areas / Monitored / Clean / Drifted / No Baseline

### Current Configuration State

Four panels: Users & Guests / Groups / App Registrations / Devices

Data is pulled fresh from Microsoft Graph at report generation time — always reflects the current state.

### Technical Appendix

Full audit trail tables: all drift results and all restore actions for the period. For internal or auditor use.

## Downloading as PDF

With a report open in the viewer, click **PDF** in the top toolbar. This opens the report in a new browser tab and triggers the browser's print dialog. Choose "Save as PDF" from your print destination.

> For best results use Chrome or Edge — they produce the cleanest PDF output from the report's print stylesheet.

---

## Downloading as Word (.docx)

Click **Download Word** in the top toolbar. TrustM365 generates a structured Word document server-side and downloads it directly — no print dialog, no intermediate steps.

The Word document is always in **light mode** — white background, dark text, professional formatting — matching the HTML report. It contains:

- Cover page: company name/logo, tenant name, period, stat summary
- All report sections as proper Word headings, body paragraphs, and formatted tables
- Alternating row shading on all data tables
- MSSP commentary rendered as indented callout blocks with accent border
- Company name and tagline in the page footer on every page

The document is generated fresh at download time — it always reflects the current database state, including any remediation since the report was originally generated.

> Company name, tagline, and report accent colour are read from MSSP Settings and applied throughout the document.

---

## Viewing saved reports

Saved reports appear in the Reports list. Each card shows:

- Report title
- Trigger (On-demand / Scheduled)
- Covered date range
- Generation timestamp
- Unread indicator (blue dot) for reports generated by the scheduler

Click any report card to open it in the viewer. Reports are stored as self-contained HTML in the database — no re-generation needed.

---

## Deleting reports

Open a report in the viewer and click **Delete** in the top-right. Reports cannot be recovered after deletion.
